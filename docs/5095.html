使用Cordova的CLI构建Mendix混合应用程序

原文：https://medium.com/mendix/mendix-hybrid-with-cordova-cli-3ef9961341d0?source=collection_archive---------0-----------------------#2021-03-04

随着Mendix使用Phonegap的时代即将结束，仍然有许多正在运行的应用程序需要维护。这意味着我们仍然需要使用混合移动应用，至少在一段时间内。因此，我写这篇博客来一步一步地解释，如何建立一个APK或IPA文件。

如何使用Cordova的命令行界面构建混合Android APK？

在Mendix上转到您的应用仪表板，并在左侧菜单中选择移动应用。参考下面图1

选择Android复选框，并确保您启用了您的应用程序可能需要访问的任何移动功能，如相机、日历或照片库。然后按“发布到移动应用商店”(图2和图3)

选择您构建应用程序的环境(在这种情况下，它是下拉菜单中的“沙盒”)，它将填充默认的免费应用程序URL，稍后可以在基于代码的中修改该URL。然后按“下载可定制的包”，看看图4

下载完成后，您将获得一个zip文件，您可以从中提取代码。(图5。)

用VScode打开名为“phonegap”的文件夹，文件夹的结构会是这样的(图6。)

在VScode中打开一个终端。首先运行下面的命令。(图7)。

npm install

在标识符处，将“-”更改为“.”，如果你不改变这一点，它可能无法建立。(图8和图9)。

如果您想要编辑任何插件，您将需要从这些文件“config.xml.mustache”和“index.html.mustache”中进行编辑(图10和图11)

然后保存所有文件，并运行命令(图12)

npm run package

您将看到，在命令完成后，文件夹构建将包含config.xml和index.html，类似于那些“小胡子”文件(图13和图14)

现在使用命令转到终端中的构建文件夹(图15)

cd build

这里你需要做的是找到安装Cordova CLI的方法，这是用这个 向导 轻松实现的。

现在运行如下所示的命令(图16)

cordova platform add android

现在在build下有一个文件夹，名为platforms/android。

在安装插件的情况下，您也可以在上述步骤之后使用Cordova CLI。(图17)

cordova plugin add cordova-plugin-customurlscheme — variable URL_SCHEME=app2

获取android文件的最后一步是运行这个命令(图18)

cordova build android

如果构建成功，可以在终端输出的最后一行所示的路径中找到APK文件。文件路径对于您的设备是唯一的(图19)。

为了将APK发布到Google Play，您需要生成一个签名版本。为此，您可以遵循本指南。

如何用Cordova命令行界面构建混合iOS

确保在选择iOS的情况下从mendix下载phonegap.zip(图20)。

转到build文件夹并运行命令(图21)。

cordova platform add ios

您还可以在这一步添加任何Cordova插件(图22)。

如果要安装任何pod依赖项，请使用以下命令转到platform/ios文件夹

cd platform/ios

然后运行命令来安装任何依赖项(图23和图24)

pod install

接下来使用XCode打开iOS文件夹(图25)

在签名和证书菜单中选择“团队”(图26)

现在，您可以在顶部访问菜单，将您的应用构建到您的iOS设备上，用于开发和测试目的，而无需将应用发布到任何在线商店(图27)。

通常情况下，您会得到以下错误:

error: The sandbox is not in sync with the Podfile.lock. Run ‘pod install’ or update your CocoaPods installation.

再次转到XCode，并为SRCROOT的构建设置中的自定义路径添加另一个配置(图28)

然后返回到构建阶段，并切换到SRCROOT(图29)

清理构建并尝试再次构建，您将看到成功消息出现(图30)

应用程序构建的结果(图31和图32)。

若要发布到Apple Store，您需要一个签名的IPA文件。为此你可以遵循这个 指南 。

请记住，发布到Apple Store时，有两个配置文件可供您选择。一个是开发，一个是发行。如果您希望您的应用程序可供公众访问，您需要选择分发配置文件。
