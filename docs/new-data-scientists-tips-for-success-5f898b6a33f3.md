# 给新数据科学家的建议

> 原文：<https://medium.com/airbnb-engineering/new-data-scientists-tips-for-success-5f898b6a33f3?source=collection_archive---------0----------------------->

在这篇文章中，我为加入 Airbnb 数据和产品团队的初级数据科学家提供建议。

你是一名新的数据科学家，恭喜你！如果你是一名初级数据科学家，在你的第一份数据工作中，你应该问自己，“我如何有效地执行？”这篇文章将集中讨论这个执行过程。它主要面向产品团队中的数据科学家，但许多技巧可以推广到任何技术职位的新员工。

在这篇文章中，我将有效执行分为以下四类:

1.  **优先级排序，**
2.  **估计任务需要多长时间，**
3.  **如何回答您的问题，以及**
4.  **交流&分享你的作品。**

# **优先排序**

![](img/a49c33e8c354616d73580b50e3949858.png)

为了更好地与经前综合症打交道，你需要对他们的工作有所了解。丹·希尔(前 Airbnb 的)在这里捕捉了一些观察。他认为项目管理创造了向最终用户交付好产品的流程。我完全同意:项目经理的技能多种多样，但*流程*通常是共同点。利用这一点。项目经理应该能够为你优先考虑他们的要求；你的经理也可以帮忙。一开始，密切关注两者的优先顺序:没有更好的方法来学习和理解什么对他们和业务是重要的！你很快就会知道事情需要多长时间。这一点，再加上影响力，将帮助你更好地确定优先顺序。

实际上，你应该有一个持续的文档来记录你的工作。对自己的工作有一个总体的了解，不仅可以让你轻松地看到自己是如何在各种类型的工作(大赌注、小赌注、临时工作、基础设施等)中度过时间的，还可以让绩效对话变得更加容易。如果你觉得你的大部分工作是为短期目标或项目服务的，你会有数据来证实它。如果是这种情况，一定要和你的经理沟通，因为他们工作的一部分就是确保你的时间花得值。这种实践还为利益相关者和业务合作伙伴提供了对您的工作流的可见性。

> Protip:你应该始终确保你的商业伙伴知道她想要什么。实际上，这意味着无论何时有人向你索要数据，你都应该确保他们清楚地说出他们需要数据的目的。通常，这个过程会让你了解到某人想要的并不总是满足他们的需求。当你在你的角色中进步时，一个有效的巨大标志是识别和回答影响决策的问题的能力。

# **估计任务需要多长时间**

作为一名新的数据科学家，了解以下两者之间的差异至关重要:

*   你*希望*某事持续多长时间(相对于你的其他优先事项和兴趣，这项任务有多令人向往)，
*   *需要*多长时间，(由于工具、基础设施、日志等因素，可行性如何)，以及
*   给定愿望和现实*，*实际需要多长时间去做。**

我注意到，初级数据科学家花费最长时间完成的任务通常不是最具智力挑战性的任务。相反，它们是定义最不明确的，或者基础设施需求没有得到充分阐述或理解的。在这些情况下，你需要争取清晰，并确保你知道你的系统能处理什么——这将有助于你和你的同事。

![](img/85e6349054fa2fac187e43e82ad99e21.png)

首先:

*   你明白分析上的要求吗？如果没有，一定要做到。当你不明白你应该调查什么或寻找什么的时候，你不可能执行得很好。
*   一旦你理解了你被要求做什么，下一步问:数据存在吗？你了解数据生成过程吗？如果日志记录或数据不存在或有问题，您就帮不上什么忙。如果是这种情况，要求添加日志记录，并明确表示任何分析都将被延迟，直到您拥有足够的数据。团队中的每个人都需要了解丢失数据的代价，不仅仅是工程师。
*   当您考虑分析选项时，临时解决方案是否合适，或者您是否应该投资于更可持续的东西？你能在将来编写一个函数或包来自动化这种类型的分析吗？尤其是在初创公司，你应该时刻想着如何扩大自己的规模。一条金玉良言:对待你的工作，就好像你将不得不重复或复制你所做的一切。

项目经理不是数据科学家，评估不同的分析方法不是他们的工作，而是你的工作。如上所述，他们的工作是提供一个交付好产品的框架。我接触过的最好的项目经理都有清晰一致的框架，尤其是在数据产品方面:对他们来说，完美和不透明(总是)是好的敌人。如果一个解决方案花费的时间是另一个的两倍，实现起来非常复杂，或者是一个黑箱，你需要非常清楚和令人信服地说明为什么它应该是首选的。你的说服力应该很少包括“AUC”或“梯度下降”这样的词。始终关注业务影响，并描述您在这些方面构建的各种数据产品/解决方案的特征！

> Protip:做主持人。很荣幸成为对数据驱动决策感兴趣的产品团队的一员。你的团队掌握的数据越多，你们的效率就越高。倡导[数据教育](/airbnb-engineering/how-airbnb-democratizes-data-science-with-data-university-3eccc71e073a)和支持数据自助服务的工具。

**如何让你的问题得到解答**

在 Airbnb，我们重视独创性和解决问题的能力，这也是我喜欢在这里工作的原因之一。但是因为我们中的许多人过分看重这些技能，所以我们经常通过展示我们的创造力来寻求帮助。你们都知道我在说什么:不是说你需要什么帮助，而是非常详细地描述你正在做什么，并询问非常具体的建议:“我如何以这种特定的方式转换数据？”、‘我如何使用[这个特定的工具]来做[这个非常特定的事情]’。我完全理解这种冲动:你在证明你正在努力，并且已经投资了一个解决方案。但是你可能错过的是，你已经琢磨过的解决方案可能是许多解决方案中的一个。当你只在某个特定的实现上寻求建议时，你已经缩小了前进的道路。当寻求帮助时(从任何人，真的)总是从目标开始；这让你接触到更广泛的输入。

> 普罗提普:在科技领域，你不可能通过垄断信息取得成功。当你得到帮助时，要养成习惯，回头和别人分享解决方案。栈溢出是一个很好的地方，但是一个[知识库](/airbnb-engineering/scaling-knowledge-at-airbnb-875d73eff091)也很棒。当信息自由流动并被广泛获取时，我们都会变得更好。

# **交流&分享你的作品**

如果你在一个嵌入式产品团队工作，交流是你工作中最重要的方面之一。在这些方面，我给初级数据科学家的最有力的建议是在不同高度进行交流的重要性。对于大多数与数据组织之外的人的交流，他们感兴趣的不是附录；是 TL；(*博士太长；不要看*。在实践中，这通常意味着你没有责任告诉业务伙伴你做了多少工作，有多努力，或者各种模型评估方法是什么——把这些讨论留给你的经理和同事吧。如果一个项目经理问了一个关于你的用户的问题，在合理的范围内，尽可能简单地回答。不要把你的回答隐藏在错综复杂的技术细节中——这样你会失去很多人！如果他们有问题(他们总是应该有)，他们会跟进。

![](img/451a5c6abb3cb35bfb16d016e42d408e.png)

你与某人合作得越多，你就越能预见他们的后续行动。但是不要假设他们对你实现目标的途径感兴趣。你的商业伙伴需要交付产品，你需要帮助他们实现这一目标。

最后，在任何关于分享你的工作的谈话中，房间里的大象是最后期限。你肯定会遇到一个商业伙伴，他会要求一些东西，但没有指定期限。当他们需要的时候却没有收到，他们会很不高兴。当你致力于项目时，一定要让他们指定并记录这些截止日期。如果你接近最后期限，并且知道你会错过它，那就主动传达。这是成熟的标志，而不是失败。

> Protip:确保尽早并经常从你的经理、其他数据科学家和业务合作伙伴那里获得对你工作的反馈。不要低估工作社会化的价值。如果你的发现是违反直觉的，或者迫使你重新考虑现有的(锚定的)数据点，这一点尤其重要。将你的工作社会化将帮助你完善、发展和传播它；在大型演示之前提出尖锐的问题要比在演示过程中好得多。最后，如果你不清楚工作社交的场所，那就去问或者自己创造。
