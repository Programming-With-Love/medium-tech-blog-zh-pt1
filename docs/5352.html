在Oracle云上使用Kafka和MySQL

原文：https://medium.com/oracledevs/go-with-kafka-mysql-on-oracle-cloud-9c93e71167c2?source=collection_archive---------0-----------------------#2017-11-27

Oracle Application Container Cloud现在支持GoT3】(golang)以及现有的运行时集，包括 Node.js、Java SE、Java EE、Python、PHP和Ruby

这个博客演示了一个基于Go的异步 worker 应用程序

  • Oracle事件中心云(托管Apache Kafka)充当消息中心
  • Oracle应用程序容器云 (polyglot，云本地开发平台)托管(Go)消费者应用程序，该应用程序使用Kafka的 sarama客户端
  • Oracle MySQL Cloud 用作存储来自消费者应用程序的已处理消息的接收器—Go MySQL驱动程序用于此目的

这里是一篇博客文章，它通过一个基于使用Redis队列中的数据的例子来讨论工人应用程序的基础

基础设施设置

Oracle事件中心云(Kafka broker)

博客中用于示例应用程序的Kafka集群拓扑相对简单，即一个代理与Zookeeper位于同一位置。

请参考文档了解更多关于拓扑和详细安装过程的细节(提示:很简单！)

创建主题— 创建主题(详细的文档)

创建自定义访问规则

您需要在Oracle Event Hub Cloud上的Kafka服务器虚拟机上创建一个自定义的访问规则来打开端口6667—详情请见。这只是帮助使用Kafka CLI进行测试的临时更改

Oracle Application Container Cloud不需要打开端口6667 (Kafka broker ),因为安全连接由服务绑定负责

Oracle MySQL云

提供一个MySQL数据库实例——您可以在这里参考详细文档

创建自定义访问规则

您需要创建一个定制的访问规则来在Oracle MySQL Cloud上的虚拟机上打开端口3306—详情请见。这只是帮助测试的临时改变

Oracle Application Container Cloud不需要打开端口3306(用于MySQL服务),因为安全连接由服务绑定负责

构建和部署

建设

  • git clone https://github.com/abhirockzz/accs-go-kafka-mysql.git
  • cd accs-go-kafka-mysql
  • zip accs-go-kafka-mysql.zip gokafkamysql.go start.sh

部署

先决条件

在部署之前，您需要用以下详细信息更新 deployment.json

  • Oracle MySQL云服务(您之前创建的)
  • Oracle事件中心云主题(您之前创建的)

借助Oracle Application Container Cloud，您在部署应用程序方面有多种选择。本博客将利用 PSM CLI ，这是一个用于管理Oracle云服务的强大命令行界面

其他部署选项包括REST APIOracle开发者云 当然还有 控制台/UI

  • 在您的机器上下载并设置PSM CLI(使用psm setup ) — 详情请点击
  • 部署应用程序— psm accs push -n accsgokafkamysql -r golang -s hourly -m manifest.json -d deployment.json -e dockerhub -p accs-go-kafka-mysql.zip

检查您的应用程序

完成后，您应该能够看到部署的应用程序及其详细信息——注意，它有两个实例，类型是 worker
