使用Eclipse集合的十大理由

你可能只需要一个。

使用 Eclipse集合的十大理由

  1. 丰富、流畅、实用的API ，具有良好的对称性(生产力)
  2. 内存效率(性能)
  3. 优化的Eager API (性能)
  4. 原始集合(生产力&性能)
  5. 不可变集合(安全&性能)
  6. 惰性和并行惰性API (性能)
  7. 多地图、袋、双地图、池、区间(产能&型安全)
  8. 散列策略数据结构(生产力&性能)
  9. 多阅读器数据结构(安全和性能)
  10. 可变和不可变集合工厂(生产力)

如果您今天使用的集合框架缺少其中的一个或多个特性，并且您需要它们，那么您真的值得花时间来检查和尝试 Eclipse集合。

尝试Eclipse集合的其他原因

  1. 它是开源的，由Eclipse Foundation管理，如果你需要什么，你可以贡献给这个库
  2. Eclipse集合 9.0 、 9.1 、 9.2 、 10.0 、 10.1 、 10.2 、 10.3 、 11.0 和 11.1 发布了许多有用的特性，这些特性是由许多开放成员贡献的
  3. 有好玩的代码和让你开始
  4. 容易理解，如果你已经知道lambdas/ streams，那么重构为
  5. Eclipse集合网站有11种不同的语言版本
