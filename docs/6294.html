自从Pinterest的Android代码库成为Kotlin的优先事项以来,已经过去了将近一年,自从我们采用了Kotlin的格式和lint的ktlint以来,已经过去了将近两年。今天,我们宣布Pinterest已正式接管该项目。

最初,我们选择ktlint是因为它的简单性,活跃的社区,可扩展性和非常敏感的所有者。它还可以轻松地与我们当前基于Phabricator的工作流程集成,并且在我们的Arcanist库中添加了大约65行PHP之后,我们能够将linting和格式应用于我们的Kotlin文件。

当 Stanley Shyiko (ktlint的开发者)几个月前打电话给我们寻找新的项目所有者时,我们立即感兴趣。它不仅是我们每天在Pinterest上使用的工具,也是我们回馈Kotlin社区的有形方式。经过与斯坦利的一些会议,很明显他完全适合,我们很高兴继续作为业主。

在短期内,项目的执行和维护方式不会发生任何变化。我们仍然欢迎并鼓励外部以问题和验证请求的形式对项目做出贡献。从中长期来看,我们计划遵循 Stanley 提出的路线图,实施一种全局禁用规则的方法(ktlint 的 最受欢迎的功能 ),整合官方的 Gradle 插件,并更新一些 ktlint 核心 API 以允许更清晰的规则编写。如果您觉得这些项目中的任何一个感兴趣,请联系或打开验证请求。最后,如果您或您的公司使用ktlint,请打开一个验证请求,将您添加到采用者列表。

我们致力于继续斯坦利开始的伟大工作,并与不断增长的Kotlin社区中的其他开发人员合作。

感谢 Garrett Moon 和 Jon Parise 协助将 ktlint 带到 Pinterest,以及来自 Google 的 Kevin Bierhoff、Beth Cutler 和 James Lau 提供技术意见。(T9 )

我们正在建造世界上第一个视觉发现引擎。全球有超过4.75亿人使用Pinterest来梦想,计划和准备他们想要做的事情。加入我们的团队!
