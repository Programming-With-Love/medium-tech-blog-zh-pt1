使用Mendix和Cmder在Windows 10中构建可插拔小部件的快速指南

原文：https://medium.com/mendix/mendix-make-a-pluggable-widget-in-window-10-543c87bbf979?source=collection_archive---------1-----------------------#2021-07-17

新开发人员可能很难在Windows 10中编译和构建他们的可插拔小工具。这就是为什么我写了这篇关于如何克服这一点的快速指南。

开始之前

我们需要一些工具来管理我们的节点版本，以及一个好的终端来执行我们的命令。

首先，你需要一个好的命令行界面。目前我用的是 Cmder 版本 1.3.18 。你可以在这里下载。

它是便携式的，意味着不需要安装。您可以在下载后提取文件，并将其保存在您的驱动器上，然后使用。exe文件来运行它。

第二个是NVM，它是一个节点模块管理工具。你可以在Github 这里下载。当然，这是一个. exe文件，只需单击“下一步”即可完成安装。

当您能够获得所有这些工具时，让我们打开Cmder并从命令行检查它。

要查看当前可用的节点LTS版本列表，您可以运行命令

nvm list available

要检查当前使用的版本，请运行命令

nvm list

要安装特定版本，例如12.22.3，您可以运行命令

nvm install 12.22.3

安装一个版本后，它不会自动选择为活动版本。要选择新安装的版本，请运行命令

nvm use 12.22.3

安装约曼

现在是门迪克斯部分。在安装Mendix小部件生成器之前，您需要安装 Yeoman 。为此，您可以运行以下命令

npm install -g yo

现在我们可以安装Mendix小部件生成器，它将创建我们的小部件支架。运行命令

npm install -g @mendix/generator-widget

为了确保它能正常工作，我们可以创建一个测试小部件，我称之为test widget。在Cmder中导航到您的Mendix应用程序的项目目录，并使用命令创建一个名为CustomWidgets的文件夹

mkdir CustomWidgets

接下来，通过运行命令导航到您刚刚创建的文件

cd CustomWidgets

我们现在可以调用之前安装的Mendix小部件生成器来创建一个测试小部件，我称之为test widget。运行命令

yo @mendix/widget TestWidget

然后完成发电机所需的所有输入。完成后，您应该会看到类似这样的内容

测试构建功能

为了确保一切按预期运行，我们可以通过删除 TestWidget 文件中的 dist 文件夹来测试构建过程，该文件是由我们之前创建的 CustomWidgets 文件中的生成器创建的。

现在运行命令

npm run build

你的新部件将会在重新创建的 dist 文件中。

完成了。
